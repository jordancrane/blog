Hosting a GitHub User Site with Project Subdomains

Posted on June 30, 2020

I started this site with the original purpose of hosting my resume for easy sharing without much else in mind. That’s still mostly all it does, but I’ve solved a few problems along the way to make deployments (slightly) cleaner, so I thought I would document my process here. I keep my resume in version control in a markdown file, and then use the fabulous Pandoc to convert it to a variety of formats, namely an HTML file for hosting and a PDF file for sharing over email.
